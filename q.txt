  1. ML/BO 部分                                                           
  
  静态 steps.json 只能描述一个固定 column 的操作。实际的 active learning  
  loop 是动态的——每列的 volumes (Vr, Vg, Vb) 由 GP 模型在运行时决定。这个
  task 文件只能作为模板（一个 column 的协议），不能替代 pipeline.py       
  的动态循环。                                              

  选择：                                                                  
  - A) 把 task 当成单列模板 + 文档，实际运行仍用 notebook/pipeline
  - B) 给 OpenOT2 task 系统加 template 支持，让 pipeline 每轮生成 steps   
                                                            
  2. CV/Camera 部分                                                       
                                                            
  task 格式没有 capture_image 这个 step kind。实际协议里 Phase 3 是：home 
  → 等待沉降 → 拍照 → 提取 RGB。目前这部分完全在 Python     
  端（pipeline.py）处理，不经过 OpenOT2 的 step 系统。                    
                                                            
  选择：                                                                  
  - A) 保持现状——camera 由 Python 控制，task 只管液体操作
  - B) 给 OpenOT2 加 capture step kind，让整个流程可以在 task 系统里编排
